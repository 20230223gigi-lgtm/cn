The Google Media API provides programmatic access to the platform's news content, enabling you to integrate articles, categories, and real-time feeds into your own applications. All endpoints follow REST conventions and return structured JSON responses, making it straightforward to build news-powered features regardless of your technology stack.

## API 基本信息

在开始集成之前，请了解以下关键参数：

| 参数 | 值 |
|------|-----|
| Base URL | `https://api.googlemedia.com/v1` |
| 协议 | HTTPS only |
| 响应格式 | JSON |
| 标准速率限制 | 100 requests/minute |
| Enterprise 速率限制 | 1,000 requests/minute |
| 访问要求 | Enterprise 订阅 |

<Note>
  API access is available exclusively on the Enterprise plan. Upgrade or review plan details at [/subscriptions/plans](/subscriptions/plans).
</Note>

## 可用端点概览

以下是所有可用 API 端点的快速参考：

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/v1/articles` | `GET` | Retrieve paginated article list |
| `/api/v1/articles/{id}` | `GET` | Retrieve a single article by ID |
| `/api/v1/categories` | `GET` | List all categories |
| `/api/v1/search` | `GET` | Full-text search across articles |
| `/api/v1/feed` | `GET` | Get personalized article feed |
| `/api/v1/webhooks` | `POST` | Register a webhook endpoint |

## GET /api/v1/feed

The feed endpoint returns a personalized stream of articles ranked for a given API key based on usage signals, subscribed categories, and preferred languages. Use this endpoint to power homepage feeds, recommendation widgets, or personalized digest emails.

### Query parameters

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `language` | string | — | Filter feed articles by language: `zh-CN`, `zh-TW`, or `en`. |
| `page` | integer | `1` | Page number for paginated results. |
| `per_page` | integer | `20` | Number of articles per page. Maximum value is `100`. |

### Example request

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://api.googlemedia.com/v1/feed?language=zh-CN&per_page=5"
```

### Example response

```json
{
  "status": "success",
  "data": [
    {
      "id": "art_11ab",
      "title": "今日要闻：全球科技峰会开幕",
      "summary": "来自50个国家的代表齐聚日内瓦...",
      "category": { "slug": "tech", "label": "科技" },
      "language": "zh-CN",
      "published_at": "2024-06-16T07:00:00Z",
      "is_premium": false
    }
  ],
  "meta": { "page": 1, "per_page": 5, "total": 280 }
}
```

## 请求格式

每个 API 请求都必须包含有效的认证头。对于携带请求体的 `POST` 请求，还需声明内容类型：

```http
Authorization: Bearer {api_key}
Content-Type: application/json
```

- **`Authorization`**：所有请求必须携带此标头，值为 `Bearer` 加您的 API 密钥。
- **`Content-Type`**：仅 `POST` 请求需要，值固定为 `application/json`。

## 响应格式

所有 API 响应均使用统一的 JSON 信封结构。`status` 字段指示请求结果，`data` 包含实际返回内容，`meta` 提供分页信息：

```json
{
  "status": "success",
  "data": { ... },
  "meta": {
    "page": 1,
    "per_page": 20,
    "total": 1540
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | string | `"success"` 或 `"error"` |
| `data` | object / array | 端点返回的核心数据 |
| `meta.page` | integer | 当前页码（从 1 开始） |
| `meta.per_page` | integer | 每页返回的条目数 |
| `meta.total` | integer | 符合条件的总条目数 |

## 错误代码

当请求未能成功完成时，API 将返回标准 HTTP 状态码以及包含错误详情的 JSON 响应体：

| HTTP 状态码 | 含义 |
|------------|------|
| `200 OK` | 请求成功，响应体包含所请求的数据 |
| `400 Bad Request` | 请求参数缺失或格式无效 |
| `401 Unauthorized` | API 密钥缺失或无效 |
| `403 Forbidden` | 密钥有效，但当前订阅计划无权访问此资源 |
| `404 Not Found` | 请求的资源不存在 |
| `429 Too Many Requests` | 超出速率限制，请稍后重试 |
| `500 Internal Server Error` | 服务器端发生意外错误，请联系支持团队 |

<Tip>
  当您收到 `429` 响应时，请检查响应头中的 `Retry-After` 字段，该字段会告知您需要等待的秒数。
</Tip>
