The search endpoint enables full-text search across the entire Google Media article archive, spanning all categories, languages, and publication dates. Results are ranked by relevance score so your users always surface the most pertinent content first. You can narrow results further with category slugs, language codes, and date-range filters — making it straightforward to power site-wide search bars, topic dashboards, or editorial discovery tools.

## GET /search

<ParamField query="q" type="string" required>
  Search query string. Supports quoted phrases for exact matches (e.g., `"贸易协议"`) and the `author:` operator to scope results to a specific writer (e.g., `author:张明`). See the note below for the full list of supported operators.
</ParamField>

<ParamField query="category" type="string">
  Limit results to a specific category slug. Use the [Categories API](/api/categories) to retrieve valid slugs.
</ParamField>

<ParamField query="language" type="string">
  Filter results by language. Accepted values: `zh-CN`, `zh-TW`, or `en`.
</ParamField>

<ParamField query="date_from" type="string">
  Return results published on or after this date. Must be an ISO 8601 date string, e.g. `2024-01-01`.
</ParamField>

<ParamField query="date_to" type="string">
  Return results published on or before this date. Must be an ISO 8601 date string, e.g. `2024-12-31`.
</ParamField>

<ParamField query="page" type="integer" default={1}>
  Page number for paginated results.
</ParamField>

<ParamField query="per_page" type="integer" default={20}>
  Number of results to return per page. Maximum value is `50`.
</ParamField>

### Example request

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://api.googlemedia.com/v1/search?q=%E4%B8%AD%E7%BE%8E%E8%B4%B8%E6%98%93&category=finance&per_page=5"
```

### Example response

```json
{
  "status": "success",
  "data": [
    {
      "id": "art_05mn",
      "title": "中美贸易谈判最新进展",
      "summary": "双方代表团于日内瓦举行...",
      "relevance_score": 0.97,
      "published_at": "2024-05-20T06:00:00Z",
      "category": { "slug": "finance", "label": "财经" }
    }
  ],
  "meta": { "page": 1, "per_page": 5, "total": 342, "query_time_ms": 48 }
}
```

Each result includes a `relevance_score` between `0.0` and `1.0`. A score closer to `1.0` indicates a stronger match against the query string. Sort your UI results by this field to surface the most relevant articles first.

<Note>
  The `q` parameter supports the following advanced operators:

  - `author:张明` — Scope results to articles written by a specific author name.
  - `"完整短语"` — Wrap a phrase in double quotes to require an exact match.
  - `category:politics` — Filter inline by category slug without using the separate `category` query parameter.

  Operators can be combined in a single query string, e.g. `author:张明 "贸易协议" category:finance`.
</Note>

<Warning>
  The search endpoint is rate-limited to **30 requests per minute** per API key — stricter than the standard 100 requests/minute limit applied to other endpoints. Exceeding this limit returns a `429 Too Many Requests` response. Implement client-side debouncing on search inputs and cache repeated queries where possible to stay within the limit.
</Warning>
